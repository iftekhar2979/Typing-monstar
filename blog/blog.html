Difference between Local storage and session Storage?

localStorage and sessionStorageare almost identical and have the same API.

local storage :

  • local storage object persisted until the clearing browser data or clearing local storage object . It means if you clear the browser data or clear the local storage data then it will clear otherwise it will persisted
  • You can create local storage by : there key will be the name of local storage object . and value can be ( object,string,number,array,array of object). The passed value must be a string if value is object then we have to create it like string and pass that on localstorage value. 
    localStorage.setItem(”key” ,"value")
  • use the getItem()method. The getItem()  method takes one argument which must be the key. This function will return the corresponding value as a string . 
    localStorage.getItem(”key” )
  • - You can delete an entry with the `removeItem()` method. The `removeItem()`  method takes one argument which will be a key of the `localStorage`  object: 
    localStorage.removeItem(”key”)

Session Storage :

  • Session storage object persisted until the tab of browser closed. If you close browser tab session storage data will clear
  • You can create session storage by : sessionStorage.setItem(”key”,”value”) there key will be the name of session storage object . and value can be ( object, string, number, array, array of object). The passed value must be a string if value is object then we have to create it like string and pass that on localstorage value. 
    sessionStorage.setItem(”key” ,"value")
  • use the getItem()method. The getItem()  method takes one argument which must be the key. This function will return the corresponding value as a string. 
    sessionStorage.getItem(”key” )
  • - You can delete an entry with the removeItem() method. The removeItem()  method takes one argument which will be a key of the sessionStorage  object. 
    sessionStorage.removeItem(”key”)

Discussion about event loop?

JS executes all operations on a single thread, but using a few smart data structures, it gives us the illusion of multi-threading. Event loop is the secret between the asynchronus Javascript.

Javascript has a run time model based on event loop. Which is responsible for executing the code , collecting proccessing event and executing queue sub tasks .

The Event Loop has one simple job — to monitor the Call Stack and the Callback Queue or Message Queue.

If the Call Stack is empty, it will take the first event from the Callback queue and will push it to the Call Stack, which effectively runs it

How many ways we can get undefined in javascript?

8 ways to get udefined:

  • A declared variable without assigning any value to it
  • Implicit returns of functions due to missing return statements.
  • return statements that do not explicitly return anything.
  • Lookups of non-existent properties in an object.
  • Function parameters that have not passed.
  • Anything that has been set to the value of undefined.
  • Any expression in the form of void(expression).
  • The value of the global variable undefined.
